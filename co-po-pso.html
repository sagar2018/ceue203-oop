Unit-wise CO Mapping with Justification

UnitTopicHrsWt (%)Mapped COJustification
1 Fundamental of Programming in Java26%CO1 Introduces Java's execution model (JVM/bytecode), Java buzzwords, and recently added features (switch expressions, records). Establishes the conceptual and historical foundation that CO1 targets.
2 Class Fundamentals414%CO1 Classes, objects, access modifiers, constructors, this keyword, nested classes, Object class methods (equals/hashCode/toString), memory management, annotations and reflection collectively build encapsulation and core OOP principles of CO1.
3 String Handling26%CO1 String, StringBuilder/StringBuffer, regular expressions, and tokenization are essential data-handling fundamentals. Mastery consolidates the programming foundation defined in CO1.
4 Inheritance, Interfaces & Packages517%CO2 (reinforces CO1) The super keyword, dynamic method dispatch, abstract classes, interfaces (functional, marker, ISP), packages, JAR files, and the Java Module System are exactly the constructs CO2 requires for modular, maintainable, reusable code. Runtime polymorphism and abstraction also reinforce CO1.
5 Exception Handling26%CO3 Checked and unchecked exceptions, try-catch-finally, throw/throws, try-with-resources, custom exceptions, and exception propagation are precisely the mechanisms CO3 targets for fault-tolerant applications.
6 Multithreaded Programming517%CO4 Thread lifecycle, Thread/Runnable, synchronization, inter-thread communication, thread priority, deadlock handling, Executor/ExecutorService, and Fork/Join map directly to CO4's goal of multi-threaded applications that optimize performance and resource utilization.
7 File NIO414%CO5 Path, Files class, file attributes, byte/character streams, serialization, and NIO (Buffer, Channel, Selector, directory streams) correspond directly to CO5's file-handling and NIO.2 capabilities for efficient file operations and data-stream processing.
8 Collection Framework & Generics620%CO6 List/Set/Map hierarchies, iteration/sorting/searching, thread-safe collections, and generics (generic classes/methods, bounded type parameters, wildcards, type erasure) map directly to CO6 for building robust, type-safe data management programs.
